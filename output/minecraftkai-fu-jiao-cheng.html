Minecraft开服教程

  • 周四 14 三月 2019
  • 码农

准备工作

运维操作具有时效性，时间久了不保证可用，请注意博文发表时间。

本文所有操作基于阿里云学生机Debian8.9系统。 费用为114人民币/年。

这里仅介绍Forgeu端。

购买服务器

服务器买不到就放弃吧。 建议阿里云新出的轻量应用服务器，给1000G的流量，相当于平均300K的带宽。 然而一般服务器大多数时间没人，就可以有效利用峰值带宽5M而不是ECS的1M。

学生机可能不能新建Debian系统，在后面可以选择重装为Debian系统。 CentOS更稳定，Ubuntu更傻瓜，我觉得Debian很折中，而且我用习惯了。

SSH入门

本地创建密钥，自己百度。 SSH密钥分为公钥和私钥，公钥相当于账号，全网随便发，私钥相当于密码，给人拷过去你就凉了。

如为新版win10推荐去应用市场装一个Debian子系统。 否则装个PUTTY也是不错的。 直接装Git然后CMD登录也不是不行。

进入轻量应用服务器控制台， 找到你刚创建的机器，后面所有的操作都要基于这个页面。

左侧，服务器运维>远程连接>密钥管理，在这把你创建的密钥的公钥copy上去。 是公钥不是私钥！

登录服务器。

终于可以贴代码了，开心。

如果本地是Windows的Debian子系统，可以这样登录

ssh root@xx.xxx.xxx.xx

这是以管理员权限登录。 推荐自己再建一个普通用户，增强安全性。 不过如果这个服务器只用于开Minecraft也无所谓了。 此处不提供新建用户的教程。

Welcome to Alibaba Cloud Elastic Compute Service !

xxxx@fajsdlkgjwelkjapsdjfkladsgh:~$

看到这些东西，说明你登录成功了。

文件下载

在JRE官网上面找到最新的linux的JRE， 右键保存链接地址，复制到wget后面，就能下载了。 要.tar.gz格式的！ 如果你了解一点Linux，你会知道Debian自带了一个JAVA。不要用这个！Debian出于安全性的考虑，软件包严重滞后。去官网下载！

找到原版的jar，下载。 可以试试MC Versions。 下载1.12.2的server jar.

下载1.12.2的forge。 在Forge官网上面找到相应的forge。 应该不用我提醒，不要下载那个installer-win吧。 下载installer。

wget https://download.oracle.com/otn-pub/java/jdk/8u201-b09/42970487e3af4f5aa5bca3f542482c60/jre-8u201-linux-x64.tar.gz?AuthParam=1552532958_d5c5dceb3a7997bdc1a01a099bb1b861
wget https://launcher.mojang.com/v1/objects/886945bfb2b978778c3a0288fd7fab09d315b25f/server.jar
wget https://files.minecraftforge.net/maven/net/minecraftforge/forge/1.12.2-14.23.5.2820/forge-1.12.2-14.23.5.2820-installer.jar

ls -alh看一下文件大小，就知道是否下载正确了。

安装JAVA

刚才下载的那个java文件名很鬼，如果你下载正确的话，是jre-8u201-linux-x64.tar.gz?AuthParam=1552532958_d5c5dceb3a7997bdc1a01a099bb1b861这个样子。 要重命名一下。

mv jre-8u201-linux-x64.tar.gz\?AuthParam\=1552532958_d5c5dceb3a7997bdc1a01a099bb1b861 jre-8u201-linux-x64.tar.gz

这回正常多了。

解压并删除安装包。

tar -zxvf jre-[Tab自动补全]
rm jre-[Tab自动补全]

因为一些原因，直接运行forge无法下载到vinella minecraft。 所以我之前手动下载了一份。 重命名。

mv server.jar minecraft_server.1.12.2.jar

安装Forge.

jre1.8.0_201/bin/java -jar forge-1.12.2-14.23.5.2820-installer.jar --installServer

删掉这个安装器。 就是这么嚣张233，因为我确定我做得没问题。

rm forge-1.12.2-14.23.5.2820-installer.jar
rm forge-1.12.2-14.23.5.2820-installer.jar.log

安装完成，启动游戏。

jre1.8.0_201/bin/java -jar forge-1.12.2-14.23.5.2820-universal.jar

可以看到启动失败了。

[11:19:57] [main/INFO] [minecraft/MinecraftServer]: You need to agree to the EULA in order to run the server. Go to eula.txt for more info.

编辑一下这个文件，把false改成true。

怎么编辑？？我并不想在这里表达我对Vim的信仰，也不想写一篇两万字的附属教程。

好的假定你编辑完了，重新启动游戏。

运行你的第一个Minecraft命令吧。

op wolfpan

我也不知道谁是wolfpan，也不是知道op是啥意思。反正网上说应该这么做。

问题来了，你不能一直在这挂着吧，一关掉本地命令行窗口，服务器就停了。这可不行。

stop

先退出服务器。

nohup jrexxxx/java -jar forgexxx.jar > console.log &

行了，打开你本地的Minecraft客户端。

Fine，你会发现还是无法登录。 可是一切都没问题呀？？

打开轻量应用服务器控制台，左侧，安全>防火墙>添加规则>自定义>25565>TCP 打开轻量应用服务器控制台，左侧，安全>防火墙>添加规则>自定义>25565>UDP

就是完全打开这个端口。
