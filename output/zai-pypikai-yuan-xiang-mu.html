注册PYPI

开源一个项目，总是要注册一个PYPI账号的。

没什么说的。欢迎访问我的PYPI主页，我叫wolfpan。

写一个项目

大概是这么个目录结构吧，

- test_project
    - test_project
        - __init__.py
    - README.md
    - setup.py
    - LICENSE

这几个文件的作用。。显而易见。

test_project/test_project目录显然是项目的根目录。 __init__文件说明这是一个包。 注意有两层文件夹，外面的并没有用，里面的才有用。

setup.py在网上有现成的，就在PYPI的教程页上。

setup.py

# setup.py
import setuptools

with open("README.md", "r") as fh:
    long_description = fh.read()

setuptools.setup(
    name="example_pkg",
    version="0.0.1",
    author="Example Author",
    author_email="author@example.com",
    description="A small example package",
    long_description=long_description,
    long_description_content_type="text/markdown",
    url="https://github.com/pypa/sampleproject",
    packages=setuptools.find_packages(),
    classifiers=[
        "Programming Language :: Python :: 3",
        "License :: OSI Approved :: MIT License",
        "Operating System :: OS Independent",
    ],
)

直接在PYPI的教程页上面搬运的。

个人做了一些小改动，把我自己的setup.py贴在这里。哦，我这个项目是django_orm_standalone。

import setuptools
import django_orm_standalone

with open('README.md', 'r') as file:
    long_description = file.read()

setuptools.setup(
    name=django_orm_standalone.__name__,
    version=django_orm_standalone.__version__,
    author=django_orm_standalone.__author__,
    author_email='haoyupan@aliyun.com',
    description=django_orm_standalone.__description__,
    long_description=long_description,
    long_description_content_type='text/markdown',
    url='https://github.com/panhaoyu/django_orm_standalone',
    packages=setuptools.find_packages(),
    classifiers=[
        'Programming Language :: Python :: 3',
        'License :: OSI Approved :: MIT License',
        'Operating System :: OS Independent',
    ],
)

上传项目

# upload.py
import os
import shutil

if os.path.exists('dist'):
    shutil.rmtree('dist')
os.system('python setup.py sdist bdist_wheel')
os.system('twine upload dist/*')

写了个小脚本，放在了test_project/upload.py。

先删除生成的文件夹，再重新生成项目，然后上传。

会要你输入你的PYPI用户名和密码。

pip安装项目

pip install your_project_name

记得先看看有没有重名的项目，这项目名可是全局唯一的哦。

如果你之前换了国内的源，那直接pip install是有延迟的，等同步之后就可以用了。

如果很急，那就

pip install --index-url=https://pypi.org/simple/ your_project_name

更新之后再次下载安装，要加上--upgrade参数。

pip install --upgrade your_project_name
